International Commodities Tracking Application

01 Overview

The client provides commodities inspection and weighing services in accordance with United States standards and regulations.

Problems

  • Outdated legacy system built on Microsoft Access with multiple disconnected applications and redundant manual data entry.
  • Lack of real-time visibility for remote users and inefficient, email-based data sharing.
  • Manual data management required to ensure regulatory compliance, with errors often forcing process restarts.
  • High risk of data inconsistencies, security vulnerabilities, and potential catastrophic data loss.

Project Goals

  • Transition from fragmented, machine-based systems to a centralized, cloud-based platform that streamlines operations.
  • Implement a secure, role-based login system to manage access and ensure controlled visibility.
  • Replace direct database interaction with guided entry forms and bulk-editing tools to reduce manual effort and errors.
  • Connect all modules through shared core data to eliminate redundancy and maintain data integrity.
  • Build a flexible configuration framework to ensure compliant data handling and document generation.
  • Build a customer-facing portal for real-time data access and digital document retrieval, replacing email-based exchanges.

Role

  • Discovery – Conduct stakeholder interviews, audit and map legacy systems, and perform comprehensive industry and competitive analyses to establish a clear understanding of user needs and business goals.
  • Strategy – Define target users and use cases, translate organizational objectives into actionable design strategies, and outline high-level user flows illustrating how key modules and features interconnect.
  • Design – Modernize the brand identity and establish a cohesive design system with clear visual language, intuitive navigation, and modular UI components that ensure scalability and consistency across the application.
  • Front-End Development – Build and refine static templates using HTML and SCSS for review and collaboration, and test React implementations to ensure design fidelity and responsiveness.
  • Refinement – Evaluate newly completed modules against existing ones, ensuring visual and experiential consistency across the evolving application.

Challenges

  • Entering an unfamiliar industry required quickly gaining a working knowledge of the shipping and commodities sectors—including terminology, regulatory frameworks, and the complete logistics process from shipment to inspection and delivery.
  • The project’s scale and tight timeline necessitated a phased approach, dividing the application into sections so that strategy, design, and development could progress concurrently. This structure enabled early developer involvement and ensured alignment with delivery milestones.
To design an intuitive and effective solution, I immersed myself in research—gaining a practical understanding of commodities, logistics, port operations, regulatory standards, and inspection processes—to modernize a legacy system long used by industry professionals.

Due to scale and timeline, the application was divided into distinct sections, with strategy, design, and development completed for each independently. As each progressed, previously completed sections were revisited and refined to maintain consistency and ensure a cohesive user experience.

Discovery

I began by reviewing stakeholder interviews to gather context and uncover initial expectations. I then explored the legacy system in depth, mapping workflows and identifying pain points from the user’s perspective. At the same time, I immersed myself in the shipping and commodities industries to understand terminology, processes, and regulatory requirements, ensuring all design decisions were informed and practical.

Strategy

By combining improvement requests from the client with insights uncovered by our team, I developed a comprehensive list of project goals and enhancements. From there, I identified key users and created detailed profiles that captured their unique needs, pain points with the existing application, and objectives for resolving those challenges. Using these insights, I then mapped high-level user flows to show how modules and features interconnect, ensuring a cohesive and intuitive overall experience.

Design

Guided by the strategy, I designed a clean, minimal application with a scalable framework capable of efficiently presenting large volumes of data and forms. The design prioritized inclusivity, offering a seamless experience for desktop users managing data and a functional, intuitive mobile experience for onsite inspectors. I developed modular, interchangeable UX components that developers could assemble to build forms, and created interactive prototypes for complex interactions, iterating with the client based on testing and feedback.

Development

From the finalized designs, I created static, styled templates that defined the overall navigation and framework, along with modular components built to strict guidelines. This approach allowed React developers to assemble screens efficiently and accurately, using hi-fidelity wireframes in Figma as a reference. Throughout the build process, I integrated accessibility considerations into the templates to preempt common usability issues. I also reviewed and refined the React implementations, ensuring that design, usability, and accessibility remained consistent across the application, ultimately delivering a cohesive and user-friendly experience.

03 Results

Solution

Final designs that translate strategy into a cohesive, intuitive experience, showcasing modular components, streamlined workflows, and a user-centered interface.

Intentionally Minimal and Mindfully Mobile

A clean, responsive interface designed for clarity, scalability, and effortless use across devices.

The application’s interface was designed with an emphasis on clarity, efficiency, and scalability. Sidebar navigation was restructured and integrated into the header to maximize usable space and support long-term growth. A clearly defined section header ensures immediate access to the home screen, search, and key user actions, streamlining daily workflows. Responsive React data tables were implemented to improve data visibility and navigation across devices, delivering a consistent and reliable user experience for both office and field users.

Seamlessly Streamlined Structure

An intuitive detail view built for efficient navigation, adaptive layouts, and rapid data entry.

The detail page features a minimal section header, allowing users to easily return to the parent data table or navigate between related entries. Custom Bootstrap tabs were engineered to gracefully degrade into a collapsible menu, optimizing usability on smaller screens. A full library of modular form components were individually designed and developed, enabling developers to assemble data entry interfaces quickly and consistently.

Thoughtful, Scalable Systems

A flexible design system of reusable components engineered for adaptability, efficiency, and lasting scalability.

The desktop framework features a modular system of interchangeable components designed for flexibility and consistency. Each element is easily editable, scalable, and visually coded to enhance usability at a subconscious level. Together, they form a unified design language that adapts fluidly to varying screen sizes while maintaining structural and aesthetic harmony.

Complex Datasets Visually Simplified

Bringing order and meaning to robust datasets through clear, user-centered presentation.

I introduced React Data Tables with custom filters to make exploring complex data faster and more intuitive. The result was a cleaner, more focused experience that reduced cognitive load and empowered users to find exactly what they needed.
